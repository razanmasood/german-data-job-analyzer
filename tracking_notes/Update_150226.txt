## What We Discovered Today 15.02.2026

### 1. **Annotation Scope Issue**
- **Problem:** We were annotating the entire job posting (description + responsibilities + benefits)
- **Solution:** Should only annotate the **Requirements/Qualifications section**
- **Impact:** This is critical - extracting from wrong sections gives misleading data

### 2. **Concatenated Words Problem**
- **Root cause:** HTML tag removal from `description` field creates issues like "AISehr gute" instead of "AI Sehr gute"
- **Your insight:** The `descriptionHtml` field in `data/raw/jobs_combined.json` likely preserves proper spacing
- **Solution:** Use `descriptionHtml` and properly parse HTML to preserve word boundaries

### 3. **Two-Step Annotation Process**
We designed a 2-step LLM annotation approach:
- **Step 1:** Extract requirements section from full job posting
- **Step 2:** Extract SKILL and TOOL entities from requirements section only

## Key Decisions Made

1. ✅ **Use LLM for section extraction** (not regex/keyword-based)
   - More robust for varied formatting
   - Handles bilingual postings better

2. ✅ **Keep practical LLM settings** 
   - `temperature=0.0`, `top_p=0.9`
   - Good enough for production, not over-engineered

3. ✅ **Preserve punctuation in requirements section**
   - Don't strip parentheses (entities often in parentheses)
   - Only fix concatenated words

4. ✅ **Store both full description AND requirements section**
   - Allows later review/debugging
   - Shows extraction quality
